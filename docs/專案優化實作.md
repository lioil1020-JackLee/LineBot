# 專案優化實作

最後更新：2026-04-07

## 1. 目標定義

你的目標是把 LINE Bot 從「可回覆」升級到「高品質、多能力、可持續優化」的助理，接近 ChatGPT / Gemini 的體驗。

產品定位補充（本次新增）：

1. 定位為「萬事通知識助理」，涵蓋生活、學習、工作、資訊整理。
2. 不提供程式碼撰寫、修改、除錯、讀碼服務。
3. 若遇到程式需求，改提供非程式碼型協助（概念解釋、學習路徑、需求梳理）。

這類產品能力可拆成 4 層：

1. 推理與回答品質：更少幻覺、更穩定結論、更清楚不確定性。
2. 記憶與個人化：跨輪次理解使用者偏好、長期任務、上下文連續性。
3. 工具與知識：RAG、網路工具、檔案理解、查證與來源引用。
4. 工程與營運：可觀測性、評估機制、安全策略、快速迭代。

本專案目前第 3 層已具備良好基礎，但第 1/2/4 層仍需補齊關鍵能力。

## 3. 與 ChatGPT/Gemini 等級差距

### 3.1 產品能力差距

1. 長期記憶不足：原本只靠短期訊息視窗。
2. 回答品質守門不足：缺少自我檢查、引用可信度評分。
3. 任務持續性不足：缺少待辦/計畫/偏好抽取與追蹤。

### 3.2 工程能力差距

1. 缺少離線評估集（eval set）與回歸評分。
2. 缺少 prompt/model 版本 A/B 評估流程。
3. 缺少內容安全策略（敏感主題、越權操作）。

## 4. 本次已實作（直接加入專案）

### 4.1 新增會話長期記憶摘要

已新增「Session Memory Summary」機制，讓 bot 可跨輪次記住使用者偏好與上下文重點。

實作內容：

1. 新增資料表 `session_memories`（摘要、同步到的訊息 id、更新時間）。
2. 新增 `SessionMemoryRepository` 供讀寫摘要。
3. `BotService` 新增能力：
   - 在回覆時，把既有記憶摘要注入 system prompt。
   - 每累積一定數量新訊息，自動呼叫模型更新摘要。
4. `app.py` 新增管理端點：
   - `GET /admin/session/{line_user_id}/memory`
5. 新增環境變數：
   - `SESSION_MEMORY_ENABLED`
   - `SESSION_MEMORY_TRIGGER_MESSAGES`
   - `SESSION_MEMORY_WINDOW_MESSAGES`
   - `SESSION_MEMORY_MAX_CHARS`
6. 測試覆蓋：新增記憶更新測試，整體測試通過。

### 4.2 為什麼先做這件事

1. 投入成本低：不需引入重量級基礎設施。
2. 立即提升體感：對話連續性、個人化回覆明顯改善。
3. 後續可擴充：可再拆分為「偏好記憶 / 任務記憶 / 事實記憶」。

### 4.3 新增回答品質守門（Phase A 已落地）

已新增「Response Guard」機制，在主回答產生後做品質評估，必要時自動重寫。

實作內容：

1. 新增 `ResponseGuardService`：
   - 以獨立審核提示評估回答品質（是否切題、是否過度自信、是否清楚）。
   - 分數低於門檻或判定不通過時，可觸發一次重寫。
2. `BotService` 已接入守門流程：
   - 主回答產生後自動 review。
   - review 建議重寫時，改用重寫版本作為最終回覆。
3. 新增環境變數：
   - `RESPONSE_GUARD_ENABLED`

4. 測試覆蓋：
   - 已新增守門器重寫生效測試，整體測試通過。

## 5. 下一步優化藍圖（建議依序）

## Phase A（1-2 週）品質守門與可觀測性

1. 回答品質守門（Answer QA Gate）
   - 產生初稿後，用第二個判斷步驟檢查：是否回答問題、是否過度自信、是否缺來源。
   - 若不達標，觸發一次重寫。
2. 引用與可信度分數
   - 對 RAG / 網頁來源標註可信度與時間。
3. 可觀測性升級
   - 每次回覆記錄：路由（RAG/Agent/Fallback）、token、延遲、錯誤碼。

## Phase B（2-4 週）記憶與任務化

1. 記憶分層
   - Profile memory：使用者偏好（語言、格式、禁忌）。
   - Task memory：使用者進行中的目標與下一步。
2. 會話目標追蹤
   - 將「要完成的事」抽取為 checklist，回覆時可提醒進度。
3. 主題摘要壓縮
   - 長對話自動壓縮，降低 token 消耗。

## Phase C（4-8 週）評估系統與安全

1. 建立 eval 資料集
   - 類別：一般問答、RAG 問答、查證、工具呼叫、多輪記憶。
2. 建立回歸評估腳本
   - 指標：正確性、完整性、引用率、拒答適切性、延遲。
3. 安全策略
   - 注入攻擊防護、越權請求拒答、敏感資訊遮罩。

## 8. 驗收標準（Definition of Done）

1. 多輪對話中，bot 能記住使用者偏好並持續套用。
2. RAG 問答可附來源，且回答內容與來源一致。
3. 對不確定問題能誠實揭露，必要時觸發外部 fallback。
4. 每次版本更新可用 eval 腳本比對品質變化。
5. 服務錯誤可在健康檢查與 logs 快速定位。
