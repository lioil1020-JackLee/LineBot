# LineBot 打包指南

本文說明如何將 LineBot 專案打包成獨立可執行檔（`.exe`）。

---

## 快速開始

### 1. 安裝 PyInstaller

```powershell
uv sync --group dev
```

這會安裝包括 PyInstaller 在內的所有開發工具。

### 2. 打包

#### onedir 模式（推薦）

```powershell
.\scripts\build.ps1
```

輸出位置：`dist/linebot-app/linebot-app.exe`

#### onefile 模式

```powershell
.\scripts\build.ps1 -OneFile
```

輸出位置：`dist/linebot-app.exe`

#### 清理舊輸出

```powershell
.\scripts\build.ps1 -Clean
```

---

## onedir vs onefile

| 特性 | onedir | onefile |
|------|--------|---------|
| 啟動速度 | 快（約 1-2 秒） | 慢（約 5-10 秒） |
| 檔案數 | 多個檔案與資料夾 | 單一 `.exe` |
| 易調試 | ✅ 易於檢視資料夾內容 | ❌ 難以檢查 |
| 防毒軟體 | 較少誤報 | 較多誤報可能性 |
| 發佈 | 推薦用於內部用途 | 推薦用於外部發佈 |
| 建議用途 | 開發、測試、內部部署 | 單檔發佈 |

## 打包後的執行

### 前置條件

1. **LM Studio 必須在執行**  
   - 啟動 LM Studio 並確保 API 在 `http://127.0.0.1:1234/v1`
   - 所需模型（例如 `qwen3-coder-30b-a3b-instruct`）已載入

2. **.env 設定檔**  
   - 將 `.env` 複製到 `.exe` 所在目錄或上層目錄
   - 確保 `LINE_CHANNEL_ACCESS_TOKEN` 和 `LINE_CHANNEL_SECRET` 已填寫

3. **資料夾權限**  
   - 執行目錄需要可寫入（用於 SQLite `data/` 資料夾）

4. **ngrok tunnel（webhook 需要）**  
   - 如果要接收 LINE webhook，另外開一個終端執行：
     ```powershell
     E:\py\LineBot\tools\ngrok\ngrok.exe http 8000 --config E:\py\LineBot\.ngrok\ngrok.yml
     ```

### 啟動方式

#### onedir

```powershell
cd dist\linebot-app
.\linebot-app.exe
```

#### onefile

```powershell
.\dist\linebot-app.exe
```

都會輸出：
```
INFO:     Will watch for changes in these directories: [...]
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

## 手動打包（不用腳本）

如果你想直接用 PyInstaller 指令：

### onedir 模式
```powershell
uv run pyinstaller linebot.spec
```

### onefile 模式
先編輯 `linebot.spec`，將 `ONE_FILE = False` 改為 `ONE_FILE = True`，再執行：
```powershell
uv run pyinstaller linebot.spec
```

或一次性指令：
```powershell
uv run pyinstaller --onefile linebot.spec
```

## 常見問題

### Q1: 打包後找不到 .env 檔案

**A**: 確保 `.env` 在 `.exe` 所在目錄或執行目錄下。  
建議方式：
- onedir 模式：複製 `.env` 到 `dist/linebot-app/` 目錄
- onefile 模式：複製 `.env` 到 `dist/` 目錄

### Q2: 防毒軟體誤報 onefile

**A**: 這是打包工具常見的問題。可：
1. 用 onedir 模式（較少誤報）
2. 向防毒軟體回報（通常是誤判）
3. 在防毒軟體中新增信任清單

### Q3: 執行時 LM Studio 連不到

**A**: 檢查：
1. LM Studio 是否真的在執行
2. `http://127.0.0.1:1234/v1` 是否能 ping 通
3. `.env` 中 `LM_STUDIO_BASE_URL` 是否正確

### Q4: 多次打包，想保留之前的版本

**A**: 改檔名或複製整個 `dist/` 資料夾再打包，因為打包會覆寫 `dist/`。

## 開發流程

### 本地開發（推薦）

不建議每次更改都重新打包。而是：

```powershell
# 第一次
uv sync
uv run linebot

# 之後修改只需重啟服務，不用重新打包
```

### 定期打包

等功能穩定後才打包發佈：

```powershell
uv run pytest  # 確保測試通過
uv run ruff check .  # 確保代碼品質
.\scripts\build.ps1  # 然後才打包
```

---

## 進階：自訂打包行為

如果要修改打包配置，編輯 `linebot.spec` 中的：

- `hidden_imports`：隱式依賴
- `datas`：要包含的資料檔案
- `binaries`：系統級依賴（如 ffmpeg）

例如，若要額外包含某個配置檔：

```python
datas = [
    (str(PROJECT_ROOT / '.env'), '.'),
    (str(PROJECT_ROOT / 'config'), 'config'),  # 新增
]
```

## 檔案結構

打包前後的關鍵檔案位置：

```
LineBot/
├── linebot.spec               # ← PyInstaller spec 檔案（核心）
├── scripts/
│   └── build.ps1              # ← 打包腳本
├── dist/
│   ├── linebot-app/           # ← onedir 輸出
│   │   ├── linebot-app.exe
│   │   ├── _internal/         # 所有依賴與資源
│   │   ├── .env               # ← 複製在此
│   │   └── data/              # ← SQLite 資料夾
│   └── linebot-app.exe        # ← onefile 輸出（若採此模式）
└── build/                     # ← 暫時產物（可刪）
```

## 後續維運

### 更新依賴後重新打包

```powershell
uv sync --group dev
.\scripts\build.ps1 -Clean  # 先清理舊的
```

### 版本管理

建議在 `linebot.spec` 中改版本號，或重新命名 `dist/` 資料夾：

```powershell
# 打包完成後
Move-Item dist\linebot-app "dist\linebot-app-v0.1.0"
```

## 需要幫助？

檢查清單：
- [ ] `.env` 已正確填寫
- [ ] LM Studio 已啟動
- [ ] `uv sync --group dev` 已執行
- [ ] `.exe` 與 `.env` 在同目錄
- [ ] 虛擬環境正常運作（本機開發用）

有問題時先檢查本機 `uv run linebot` 是否正常運作，再嘗試打包版本。
