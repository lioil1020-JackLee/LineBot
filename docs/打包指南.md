# LineBot 打包指南

本文說明如何把專案打包成 Windows 可執行檔。

> Windows 打包版行為：主視窗按最小化會縮到系統列（通知區域），托盤圖示使用 `lioil.ico`。

## 打包前檢查

```powershell
uv sync --group dev
uv run pytest -q
uv run ruff check .
```

## 使用 uv 打包（建議）

### onedir（預設，較穩定）

```powershell
uv run build-onedir
```

輸出：

- `dist/linebot-app/linebot-app.exe`

### onefile（單檔發佈）

```powershell
uv run build-onefile
```

輸出：

- `dist/linebot-app.exe`

### 清理舊產物

```powershell
uv run python -m linebot_app.build_tools --mode onedir --clean
```

## 重要檔案

- `linebot-onedir.spec`
- `linebot-onefile.spec`

## 舊版腳本

`scripts/build.ps1` 屬於舊流程，可保留相容性，但新建議改用 `uv run build-onedir` / `uv run build-onefile`。

若需要同時控制模式與清理，可使用：

```powershell
uv run python -m linebot_app.build_tools --mode onedir --clean
uv run python -m linebot_app.build_tools --mode onefile --clean
```

## 執行需求

1. LM Studio 可連線（通常是 `http://127.0.0.1:1234/v1`）。
2. `.env` 有正確的 LINE 憑證。
3. 執行目錄可寫入（資料庫與 runtime 檔案）。

## 常見問題

### 找不到 `.env`

請將 `.env` 放在執行目錄（或可被程式讀到的位置）。

### onefile 被防毒誤報

這是常見現象，建議優先用 onedir 發佈內部版本。

### 打包後啟動慢

onefile 會先解壓暫存資源，通常比 onedir 慢。

## 建議流程

1. 開發階段只跑 `uv run linebot`。
2. 發版前才打包。
3. 打包後做一次 smoke test：`/health`、`/health/detail`、LINE webhook。
