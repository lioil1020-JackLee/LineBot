# LineBot 打包指南

本文說明如何把專案打包成 Windows 可執行檔。

## 打包前檢查

```powershell
uv sync --group dev
uv run pytest -q
uv run ruff check .
```

## 一鍵打包（建議）

### onedir（預設，較穩定）

```powershell
.\scripts\build.ps1
```

輸出：

- `dist/linebot-app/linebot-app.exe`

### onefile（單檔發佈）

```powershell
.\scripts\build.ps1 -OneFile
```

輸出：

- `dist/linebot-app.exe`

### 清理舊產物

```powershell
.\scripts\build.ps1 -Clean
```

## 重要檔案

- `linebot_clean_onefile.spec`
- `scripts/build.ps1`

## 手動打包（不經腳本）

```powershell
uv run pyinstaller linebot_clean_onefile.spec
```

若要切換 onedir / onefile，可編輯 spec 內的 `ONE_FILE`。

## 執行需求

1. LM Studio 可連線（通常是 `http://127.0.0.1:1234/v1`）。
2. `.env` 有正確的 LINE 憑證。
3. 執行目錄可寫入（資料庫與 runtime 檔案）。

## 常見問題

### 找不到 `.env`

請將 `.env` 放在執行目錄（或可被程式讀到的位置）。

### onefile 被防毒誤報

這是常見現象，建議優先用 onedir 發佈內部版本。

### 打包後啟動慢

onefile 會先解壓暫存資源，通常比 onedir 慢。

## 建議流程

1. 開發階段只跑 `uv run linebot`。
2. 發版前才打包。
3. 打包後做一次 smoke test：`/health`、`/health/detail`、LINE webhook。
